# Asset Capture Guide

Use `assets/` for implementation screenshots and short recordings once a concrete runtime exists.

Suggested captures:

- `call-timeline.png`: session timeline with partial/final transcript markers
- `handoff-transition.png`: escalation state transition with target queue metadata
- `postcall-summary.png`: generated summary payload and delivery status
- `latency-breakdown.png`: p50/p95 timing breakdown for ASR, policy, tool call, and TTS start

Keep any screenshots scrubbed of personal data and account identifiers.
